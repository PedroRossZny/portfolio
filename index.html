O projeto foi criado para resolver um problema real enfrentado diariamente no setor da saúde: a desorganização no fluxo de atendimento, a ausência de histórico estruturado e a falta de transparência nas chamadas.

Sistema próprio com interface dedicada, geração de relatórios em PDF, controle inteligente de chamadas e personalização visual conforme a identidade da unidade.

Extensão para e-SUS:

Integração via navegador que permite realizar chamadas diretamente do ambiente oficial, sem modificar o banco de dados do sistema governamental, funcionando como uma camada complementar segura.

Entre os principais diferenciais estão:

  • Controle de múltiplas chamadas (1ª, 2ª e 3ª tentativa com diferenciação visual)
  • Sistema anti-spam para evitar múltiplos cliques
  • Registro completo com horário, sala e profissional responsável
  • Geração de relatórios de fluxo
  • Exibição em painel digital com alerta sonoro
  • Estrutura escalável e preparada para expansão

O projeto foi desenvolvido com foco em desempenho, segurança e escalabilidade, utilizando Next.js, NestJS e PostgreSQL. O tempo médio de envio das informações ao painel é de aproximadamente 0,3 segundos em conexões convencionais.

Mais do que um sistema de chamadas, trata-se de uma ferramenta estratégica para organização, transparência e eficiência no atendimento em unidades de saúde.

O projeto utiliza React Navigation para roteamento, AsyncStorage para persistência local de dados e integração com push notifications para atualização do status dos pedidos.

Aplicativo mobile desenvolvido com React Native utilizando Expo, simulando o funcionamento de um sistema de pedidos para uma cafeteria digital.

O projeto inclui um fluxo completo de usuário, iniciando com tela de splash, autenticação e criação de conta, permitindo cadastro com nome, e-mail, data de nascimento, endereço e senha. Os dados dos usuários são armazenados localmente utilizando AsyncStorage, permitindo persistência de múltiplas contas e gerenciamento de sessão.

Após o login, o usuário pode acessar o cardápio interativo, que apresenta produtos com imagem, descrição e preço. Os itens podem ser adicionados ou removidos de um carrinho de compras dinâmico, com cálculo automático do valor total do pedido.

O aplicativo também inclui um sistema de pagamento simulado, oferecendo opções de Pix ou cartão, com validação básica de dados quando a opção de cartão é selecionada. Após a confirmação do pagamento, um número de pedido é gerado automaticamente, simulando um fluxo real de compra.

Além disso, o projeto utiliza push notifications para enviar mensagens ao usuário, como notificações de boas-vindas e avisos quando o pedido está pronto para retirada.

O app também possui uma área de perfil do usuário, permitindo visualizar e editar informações pessoais, além de atualizar a senha mediante validação da senha atual. Toda a navegação entre telas é gerenciada utilizando React Navigation com Stack Navigator, organizando o fluxo completo da aplicação.

O projeto demonstra conceitos importantes de desenvolvimento mobile, incluindo gerenciamento de estado com React Hooks, persistência de dados local, navegação entre múltiplas telas, validação de formulários, simulação de fluxo de e-commerce e integração com notificações do sistema.
